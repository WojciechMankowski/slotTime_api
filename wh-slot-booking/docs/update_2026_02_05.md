## 1. Decyzje architektoniczne (zatwierdzone)

### 1.1 Frontend
- ✅ Zostajemy przy **React**
- ❌ Nie używamy Redux / Context / custom hooków
- ✅ Zasada: **1 plik = 1 ekran**
- ✅ React używany jako "dynamiczny HTML + fetch"

### 1.2 Backend
- ✅ FastAPI zostaje
- ✅ API jest **jedynym źródłem prawdy** (UI nie waliduje duplikatów)
- ✅ Jeden endpoint `/api/slots/generate`:
  - ręczne tworzenie
  - masowe generowanie
  - brak duplikatów
  - edycja slotów (tylko Admin i Super-Admin, edycja tylko dla dnia obecnego i dni następnych)

---

## 2. Struktura ekranów (docelowa)

### 2.1 Slots.tsx
**Cel:** przegląd + akcje

Zawiera:
- filtr dat (od / do)
- tabelę slotów
- akcje zależne od roli (admin / super-admin / client / viewer)
- widok kalendarza (oś pozioma to Data a oś pionowa to godziny; tu też działają akcje zależne od roli)

Nie zawiera:
- tworzenia slotów (jeżeli client lub viewer; reszta może edytować tylko jeżeli na slocie nie ma akcji/rezerwacji)
- generowania

---

### 2.2 GenerateSlots.tsx (PRIORYTET)
**Cel:** tworzenie slotów

Sekcje:

#### A. Ręczne dodanie slotów
- data
- start time
- end time
- interwał (min) **obowiązkowy**
- typ slotu
- parallel slots
- przycisk: **Utwórz sloty**

Wywołuje:

---

#### B. Masowe generowanie (z szablonu)
- select szablonu
- data od
- data do
- parallel slots (opcjonalnie override)
- przycisk: **Generuj sloty**

---

#### C. Raport (po ostatniej operacji)
- per dzień:
  - requested
  - generated
  - skipped

---

### 2.3 Templates.tsx
**Cel:** definicja dnia pracy (SZABLONY)

Zawiera:
- listę istniejących szablonów
- Generator Szablonów (D):

[D] Generator Szablonów
────────────────────────────────────
Szablon: [ ▼ Nazwa / Nowy ]
Start: [ 08:00 ]
Koniec: [ 16:00 ]
Interwał (min): [ 30 ]
Typ slotu: [ INBOUND | OUTBOUND | ANY ]
Parallel slots: [ (puste = domyślne) ]

[ Zapisz szablon ] [ Usuń szablon ]



Możliwości:
- tworzenie nowego szablonu
- edycja istniejącego
- usunięcie szablonu

Nie zawiera:
- generowania slotów
- raportów

## 3.1 Doprecyzowania funkcjonalne (2026-02-05)

### Raport [C]
- ✅ Raport dotyczy **ostatniej operacji generowania**
- ✅ Użytkownik **sam wybiera zakres dat** (od / do)
- ❌ Nie prowadzimy historii operacji (brak „ostatnie 7 dni”)

### Sekcja [A] – Ręczne dodanie slotów
- ✅ Pole **interwał jest obowiązkowe**
- ❌ Brak automatycznego wyliczania (end - start)
- Powód: jawność + brak magii (ADHD-friendly)

### Role – Super Admin
- ✅ Super Admin jest **realną rolą logiczną**
- ✅ Widzi wszystkie konta: Admin / Client / Viewer
- ❌ Na razie **brak różnic UI** względem Admina
- UI różnice mogą pojawić się później
